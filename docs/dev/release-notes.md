# 发布说明

## 2026-03-19 时区契约调整

- 新增 `APP_TIMEZONE` 配置项，默认值为 `Asia/Shanghai`。
- 数据库存储语义保持不变：所有 `DateTime` 字段仍然保存 UTC 语义的 naive datetime。
- 所有对外 API 时间字段改为输出带显式偏移的 ISO 8601 字符串，例如 `2026-03-19T08:36:23+08:00`。
- 新增只读接口 `/api/app-config`，前端任务卡片、日志卡片、时间线按天分组、编年史视图会在启动时同步 `APP_TIMEZONE` 后再渲染时间。
- Markdown 导出与应用日志时间同步切到 `APP_TIMEZONE`，避免与 UI 出现跨日偏差。

## 升级注意事项

- 外部 API 消费方如果之前默认把时间当作无时区字符串处理，需要改为读取显式偏移。
- 不需要对历史数据库记录做批量 `+8h` 迁移；旧记录会在读取时自动转换到 UTC+8 展示。
