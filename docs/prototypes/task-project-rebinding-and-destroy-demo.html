任务项目改绑与销毁原因

这个原型聚焦两个规则：任务处于 Backlog 时可以改绑定项目； 一旦任务开始，项目立即锁定，只能走“销毁任务 + 填写原因”的回收路径。

Backlog 允许改绑定项目
Started 项目锁定 + 允许销毁
Destroyed 保留原因与时间线审计
