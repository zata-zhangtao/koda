确认 Complete 后，如何避免“其实已收口但 UI 还在转圈”

这个原型对比两条路径：左边不是“后端还没完成”，而是 前端把手动完成成功和全量刷新绑定得过深； 右边则展示目标修复态：一旦 `POST /manual-complete` 返回 200， 立即以返回的 Task 快照更新界面，再把 dashboard 全量刷新降级为后台同步。

阅读重点
  • 先点击“查看完成检查单”，再点击“确认 Complete”。
  • 在“当前问题态”里，后端已成功收口，但按钮仍被 dashboard 刷新卡住。
  • 在“目标修复态”里，按钮忙碌态会在 API 成功后立即释放，页面同步切到 Completed。
