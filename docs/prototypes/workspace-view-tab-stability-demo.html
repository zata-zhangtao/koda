顶部 Workspace Tabs 切换不该再“先空白，再回填”。

这个原型专门用来评审 `ACTIVE / COMPLETED / CHANGES` 头部 tab 的切换稳定性。上方控制条驱动两个并排面板：左侧模拟当前实现中容易出现的闪动，右侧展示目标方案，要求详情区域在切换时保持连续感，不再先清空再回填。

本页关注两个变化

第一，tab 点击后详情区域不能立即被空状态占位；第二，tab 按钮自身在计数刷新时要保持宽度稳定，避免 pill 视觉抖动。

  • 当前模拟 点击 tab 后先显示 loading 占位，再渲染列表与详情，代表“状态重置过早”。
  • 目标模拟 先保留旧详情，再在新数据 ready 后平滑替换，代表“状态切换与副作用解耦”。
  • 验证重点 连续点击不同 tab 时，右侧面板不允许出现整块空白，也不允许 tab 胶囊宽度跳变。
Current Behavior 切换后先 blank，再回填列表和详情，容易在高频点击时被感知为“闪动”。
