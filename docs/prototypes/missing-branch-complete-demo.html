完成检查单

用户需要先看一眼这些确认项，系统才允许“确认 Complete”。 这能避免把“分支缺失”误判成“肯定已完成”。

  1. 检查最近一次时间线记录，确认需求已经进入可交付阶段。
  2. 必要时点“打开项目”或“打开 Worktree”查看本地代码现状。
  3. 确认分支缺失是人工 merge/cleanup 的结果，而不是误删或未提交变更。
  4. 确认后再把需求切到 `Complete / Done`，并写入一条人工确认日志。
